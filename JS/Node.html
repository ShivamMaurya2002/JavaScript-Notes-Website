Basics of Node.js

Node.js is a JavaScript runtime environment that allows you to run JavaScript on the server, not just in the browser. It is widely used for building backend applications because it is fast, efficient, and scalable. Let’s explore its basics.

1. What is Node.js?

Node.js is built on the **V8 JavaScript Engine** (the same engine used by Google Chrome) and allows JavaScript to interact with the system. It’s especially good for real-time applications like chat apps and APIs.

Think of Node.js as a way to run JavaScript outside the browser.

2. Non-blocking and Asynchronous

Node.js uses a **non-blocking, event-driven architecture**, meaning it can handle multiple requests at the same time without waiting for one to finish before starting another. 


// Example of asynchronous behavior
setTimeout(() => {
    console.log("This happens later!");
}, 1000);

console.log("This happens first!");

This shows how Node.js handles multiple tasks efficiently.

3. Modules in Node.js

Modules are reusable pieces of code. Node.js has a collection of built-in modules (like fs, http) and allows you to create your own.

Built-in Module Example

The fs module is used to interact with the file system. 


const fs = require('fs');
fs.writeFileSync('example.txt', 'Hello, Node.js!');

This creates a file called example.txt with the content "Hello, Node.js!"

Creating Your Own Module

You can create your own modules to organize your code. 


// myModule.js
module.exports.sayHello = function() {
    console.log("Hello from my module!");
};

// main.js
const myModule = require('./myModule');
myModule.sayHello();

4. npm (Node Package Manager)

**npm** is a tool that comes with Node.js and is used to install libraries and packages. You can find thousands of pre-built tools to speed up your development. 


// Installing a package
npm install express

This installs the popular express package for building web servers.

5. Creating a Basic Server

Node.js is great for creating servers. Here’s an example of a simple server using the http module: 


const http = require('http');

const server = http.createServer((req, res) => {
    res.write('Hello, World!');
    res.end();
});

server.listen(3000, () => {
    console.log('Server is running on http://localhost:3000');
});

This starts a server on http://localhost:3000 that says "Hello, World!"

6. Event-Driven Architecture

Node.js works on events. For example, you can listen for a specific action and run code when it happens. 


const EventEmitter = require('events');
const event = new EventEmitter();

event.on('sayHello', () => {
    console.log('Hello, Event!');
});

event.emit('sayHello');

This logs "Hello, Event!" when the event sayHello is triggered.

Conclusion

Node.js makes it possible to use JavaScript for backend development. By learning its non-blocking architecture, modules, npm, and basic server creation, you can start building fast and scalable applications.
