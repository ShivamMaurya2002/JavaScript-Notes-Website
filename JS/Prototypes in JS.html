Prototypes in JavaScript

Prototypes in JavaScript are a way to share properties and methods between objects. Every object in JavaScript has a built-in property called __proto__ (or its modern equivalent, Object.getPrototypeOf()), which links it to another object. This is known as the prototype chain.

How Prototypes Work

Prototypes allow objects to inherit features from other objects. If a property or method is not found on an object, JavaScript looks for it on the object's prototype, and so on, up the prototype chain.

const animal = {
    eat: function() {
        console.log("I am eating!");
    }
};

const dog = Object.create(animal); // dog inherits from animal
dog.bark = function() {
    console.log("I am barking!");
};

dog.bark(); // Output: I am barking!
dog.eat();  // Output: I am eating!

Output: I am barking! I am eating!

Adding Properties to Prototypes

You can add properties or methods to a prototype, and all objects inheriting from it will automatically have access to them: 


function Person(name) {
    this.name = name;
}

Person.prototype.greet = function() {
    console.log("Hello, my name is " + this.name);
};

const alice = new Person("Alice");
const bob = new Person("Bob");

alice.greet(); // Output: Hello, my name is Alice
bob.greet();   // Output: Hello, my name is Bob

Output: Hello, my name is Alice | Hello, my name is Bob

Prototype Chain

JavaScript uses the prototype chain to look for properties and methods. If a property is not found on the object, it checks the prototype, and so on until it reaches null. 


const vehicle = {
    drive: function() {
        console.log("Driving...");
    }
};

const car = Object.create(vehicle);
car.honk = function() {
    console.log("Honking...");
};

car.honk();   // Output: Honking...
car.drive();  // Output: Driving...
console.log(car.toString()); // Output: [object Object], inherited from Object.prototype

Output: Honking... Driving... [object Object]

Key Points About Prototypes

  • Prototypes enable inheritance in JavaScript.
  • Every object has a prototype, except the base object (null).
  • Adding methods or properties to a prototype affects all objects that inherit from it.

Conclusion

Prototypes are a powerful feature of JavaScript that enable code reuse and inheritance. Understanding how prototypes work and how to use them effectively can make your code more efficient and easier to maintain.
