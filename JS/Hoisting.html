What is Hoisting in JavaScript?

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compile phase. This means you can use variables or call functions before they are declared in the code.

Note that only declarations are hoisted, not initializations. Variables declared using let and const are hoisted but remain in a "temporal dead zone" until they are initialized.

Examples of Hoisting

1. Variable Hoisting

Variables declared using var are hoisted and initialized with undefined. 


// Example of var hoisting
console.log(myVar); // Output: undefined
var myVar = "Hello, Hoisting!";
console.log(myVar); // Output: Hello, Hoisting!

Variables declared with let or const are hoisted but not initialized. 


// Example of let/const hoisting
console.log(myLet); // ReferenceError: Cannot access 'myLet' before initialization
let myLet = "Hello, Let!";

2. Function Hoisting

Function declarations are fully hoisted, so they can be called before their definition. 


// Example of function hoisting
greet(); // Output: Hello, World!

function greet() {
    console.log("Hello, World!");
}

Function expressions are not hoisted. 


// Example of function expression hoisting
sayHi(); // TypeError: sayHi is not a function
var sayHi = function() {
    console.log("Hi!");
};
