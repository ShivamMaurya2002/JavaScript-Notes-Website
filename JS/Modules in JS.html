Modules in JavaScript

JavaScript modules allow you to divide your code into separate files, each containing reusable functions, variables, or objects. These modules help improve code maintainability and readability, especially in large projects. They can be imported or exported as needed.

1. Exporting in JavaScript Modules

JavaScript provides two ways to export functionalities from a module: default export and named export.

Default Export

A default export is used when you want to export a single value or function from a module. It can be imported without using curly braces. 


export default function greet() {
    return 'Hello!';
}

Export a single default function or value.

Named Export

A named export allows you to export multiple values from a module. They must be imported using their exact names enclosed in curly braces. 


export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

Export multiple values using named exports.

2. Importing in JavaScript Modules

Modules can be imported using the import statement, which allows you to use the exported functionalities from another file.

Importing Default Exports


import greet from './greetings.js';
console.log(greet()); // Output: Hello!

Import a default export without curly braces.

Importing Named Exports


import { add, subtract } from './math.js';
console.log(add(2, 3)); // Output: 5
console.log(subtract(5, 3)); // Output: 2

Import specific named exports using curly braces.

3. Combining Exports

A module can combine both default and named exports. This allows flexibility in exporting multiple functionalities from a single file. 


export default function multiply(a, b) {
    return a * b;
}
export const add = (a, b) => a + b;

Combine default and named exports in a module.

4. Benefits of JavaScript Modules

  • Improves code organization and readability.
  • Encourages reuse of code across different parts of an application.
  • Supports dependency management.
  • Prevents global namespace pollution.

Conclusion

Modules in JavaScript are a powerful feature that helps developers structure and manage their code efficiently. By using exports and imports, you can create modular and reusable code, making your projects scalable and maintainable.
