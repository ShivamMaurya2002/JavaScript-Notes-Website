Types of Errors in JavaScript

JavaScript has several types of errors that can occur during the execution of code. Understanding these errors helps in debugging and improving code quality.

1. Syntax Errors

These occur when the JavaScript parser encounters code that does not follow the correct syntax. 


// Syntax Error Example:
console.log("Hello World;

Output: SyntaxError: missing ) after argument list

2. Reference Errors

These occur when trying to access a variable that has not been declared or is out of scope. 


// Reference Error Example:
console.log(myVar); // myVar is not defined

Output: ReferenceError: myVar is not defined

3. Type Errors

These occur when an operation is performed on a value of the wrong type. 


// Type Error Example:
let num = 5;
num.toUpperCase(); // Trying to call a string method on a number

Output: TypeError: num.toUpperCase is not a function

4. Range Errors

These occur when a value is not within the expected range. 


// Range Error Example:
let num = 1;
num.toFixed(100); // toFixed expects a number between 0 and 20

Output: RangeError: toFixed() digits argument must be between 0 and 20

5. Eval Errors

These occur when the eval() function is used improperly. (Though rarely encountered in modern JavaScript.) 


// Eval Error Example:
eval("alert('Hello"); // Missing closing quote

Output: EvalError: Unterminated string constant

Conclusion

Each error type provides useful debugging information, allowing developers to identify and resolve issues efficiently.
