Difference Between var and let Keywords in JavaScript

Both var and let are used to declare variables in JavaScript, but they differ in several key aspects, including scoping rules and hoisting behavior.

1. Scope

One of the major differences between var and let is their scope:

  • var: The variable declared with var has a function scope or global scope (if declared outside a function).
  • let: The variable declared with let has a block scope, meaning it is limited to the block, statement, or expression in which it is used.

Example:

        
function example() {
    if (true) {
        var x = 5;  // var has function scope
        let y = 10; // let has block scope
    }
    console.log(x);  // Output: 5 (x is accessible)
    console.log(y);  // Error: y is not defined

2. Hoisting

Both var and let are hoisted, but there is a significant difference:

  • var: Variables declared with var are hoisted to the top of their scope and initialized with undefined.
  • let: Variables declared with let are hoisted, but they are not initialized until the code execution reaches the declaration. This results in a "temporal dead zone" (TDZ) between the hoisting and initialization.

Example:

        
console.log(a);  // Output: undefined (due to hoisting)
var a = 5;

console.log(b);  // Error: Cannot access 'b' before initialization
let b = 10;

3. Re-declaration

Another key difference is whether you can redeclare a variable in the same scope:

  • var: You can redeclare a variable declared with var within the same scope without causing an error.
  • let: You cannot redeclare a variable declared with let in the same scope. Doing so will throw a syntax error.

Example:

        
var x = 5;
var x = 10; // No error (var allows re-declaration)

let y = 15;
let y = 20; // Error: Cannot redeclare block-scoped variable 'y'

Summary

Aspect var let
Scope Function scope or global scope Block scope
Hoisting Hoisted and initialized with undefined Hoisted but not initialized (temporal dead zone)
Re-declaration Allowed in the same scope Not allowed in the same scope

In summary, let is generally preferred over var for variable declarations due to its block scope and more predictable behavior.
